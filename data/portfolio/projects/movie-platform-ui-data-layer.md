# Movie Platform UI and Data Layer

## Summary
A responsive React and TypeScript dashboard for managing movie data, built with CRUD operations, advanced filtering, sorting, pagination, and runtime switching between GraphQL and REST APIs.

## Problem
Frontend applications need flexible data-access patterns that can support multiple API styles while still giving users fast, reliable filtering, search, and editing workflows.

## Responsibilities
- Built the responsive React dashboard
- Integrated GraphQL and REST APIs
- Implemented runtime switching between data-fetching strategies
- Designed table and form workflows for CRUD operations
- Supported PostgreSQL-backed search and partial update behavior

## Technical Details
- React
- TypeScript
- Vite
- TanStack Query
- TanStack Router
- GraphQL
- REST APIs
- PostgreSQL
- Docker
- pg_trgm fuzzy search
- GIN indexing
- JSONB partial updates

## Notable Work
- Built CRUD workflows for creating, reading, updating, and deleting movie records.
- Added advanced filtering, sorting, and pagination for large data sets.
- Integrated both GraphQL and REST APIs behind runtime switching.
- Used PostgreSQL functions for fuzzy search, indexed queries, and partial JSONB updates.

## Outcomes
- Demonstrated practical comparison of GraphQL and REST data-fetching approaches.
- Improved frontend flexibility by decoupling UI workflows from a single API style.
- Delivered a responsive, data-rich dashboard experience.
